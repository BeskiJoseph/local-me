================================================================================
RANKING WEIGHTS
================================================================================

recencyScore (50%):
  1 / (1 + hoursOld)
  Newer = higher

engagementScore (30%):
  (likes + comments*2 + views*0.1) / 100
  More interactions = higher

distanceScore (20%):
  1 / (1 + distanceKm)
  Closer = higher

FINAL = recency*0.5 + engagement*0.3 + distance*0.2

================================================================================
KEY SERVICES (Backend)
================================================================================

FeedService.getHybridFeed():
  1. queryLocalFeed(40)
  2. queryGlobalFeed(40)
  3. merge([local, global])
  4. deduplicatePosts(merged, seenIds)
  5. computeScores(dedupedPosts, userLocation)
  6. sort by score DESC
  7. limit(20)
  8. return {posts, nextCursor, hasMore, seenIds}

PostRepository.queryGlobalFeed():
  - Query Firestore posts collection
  - Filter: status='active'
  - Sort: createdAt DESC, __name__ DESC
  - Limit: pageSize+1
  - Use cursor for pagination

PostRepository.queryLocalFeed():
  - Query by geohash bounds
  - Filter: status='active'
  - Sort: createdAt DESC, __name__ DESC
  - Limit: pageSize+1
  - Use cursor for pagination

PostController.getHybridFeed():
  - Parse cursor from query/body
  - Validate user location
  - Get user context (likes, follows)
  - Call feedService.getHybridFeed()
  - Return standardized response

================================================================================
FIRESTORE INDEXES
================================================================================

Posts Collection Indexes:
  1. createdAt DESC (primary feed sort)
  2. scope + createdAt DESC (local feed)
  3. latitude, longitude, createdAt DESC (geo queries)
  4. authorId + createdAt DESC (author posts)
  5. category + createdAt DESC (category filter)
  6. status + createdAt DESC (active posts)

Geohash:
  - 9-character precision (~22 meters)
  - Used for range queries with bounds
  - Indexed for efficiency

================================================================================
CRITICAL DECISION POINTS
================================================================================

1. Ranking ONLY on merged batch (safe scope)
   NOT on entire Firestore collection (expensive)

2. Cursor pagination NOT offset
   Prevents duplicates/skips on live data

3. Hard dedup before limit
   NOT after limit (ensures diversity)

4. Ranking score NOT used for cursor
   Use createdAt+postId for stable pagination
