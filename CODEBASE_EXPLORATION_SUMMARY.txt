4. FEED ENGINE ARCHITECTURE (Phase 4)
================================================================================

BACKEND HYBRID FEED PIPELINE (8 Steps):

1. FETCH LOCAL POSTS (pageSize * 2)
   └─ Query Firestore with geohash bounds

2. FETCH GLOBAL POSTS (pageSize * 2)
   └─ Query Firestore global collection

3. MERGE SOURCES
   └─ [local posts] + [global posts] = ~80 posts

4. DEDUPLICATE
   └─ Remove posts already seen by user via seenIds

5. COMPUTE SCORES (Phase 4 - lightweight heuristic)
   ├─ Recency Score (50% weight): 1/(1 + hoursOld)
   ├─ Engagement Score (30%): (likes + comments*2 + views*0.1) / 100
   └─ Distance Score (20%): 1/(1 + distanceKm)

6. RANK BY SCORE
   └─ Sort merged posts by computed score (descending)

7. LIMIT TO PAGE SIZE
   └─ Take top 20 posts after ranking

8. BUILD CURSOR & RETURN
   └─ nextCursor = {createdAt, postId, authorName}

RANKING WEIGHTS:
  - Recency (50%): Newer posts prioritized
  - Engagement (30%): Likes, comments, views
  - Distance (20%): Geographic proximity

FEED TYPES:
  - global: All users, worldwide scope, createdAt DESC + ranking
  - local: Users within geohash, geographic bounds
  - hybrid: local + global merged with ranking
  - filtered: By author/category/city
  - explore: Trending posts, engagement score DESC

5. DATABASE STRUCTURE (Firestore)
================================================================================

Posts Collection:
  posts/{postId}
    ├── authorId: string
    ├── authorName: string
    ├── authorProfileImage: string
    ├── title: string
    ├── body: string
    ├── text: string (indexed)
    ├── category: string (indexed)
    ├── mediaUrl: string
    ├── mediaType: string (image|video|none)
    ├── scope: string (indexed) [local|global]
    ├── latitude: number (indexed)
    ├── longitude: number (indexed)
    ├── geoHash: string (indexed for geo queries)
    ├── createdAt: timestamp (indexed DESC)
    ├── updatedAt: timestamp
    ├── likeCount: number
    ├── commentCount: number
    ├── viewCount: number
    ├── status: string [active|deleted|archived]
    ├── isEvent: boolean
    ├── eventStartDate: timestamp
    ├── eventEndDate: timestamp
    └── attendeeCount: number

KEY INDEXES:
  - createdAt DESC (primary sort for feed)
  - scope + createdAt DESC (local feed)
  - latitude, longitude, createdAt DESC (geo queries)
  - authorId + createdAt DESC (author posts)
  - category + createdAt DESC (category filter)
